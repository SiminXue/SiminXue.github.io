【文献阅读】Detecting and Mitigating Secret-Key Leaks in Source Code Repositories

0x00 摘要

【整段翻译】
去年一些新闻文章强调，恶意用户会窃取托管于公共资源仓库的文件中嵌入的API keys，以达到驱动自己的工作负荷的目的。
一些服务提供商，比如亚马逊，已经开始设法通过搜索公共仓库和暂停使用泄露的API keys来发现这些开发者的粗心大意。
但是，却基本没有从源码共享平台跟踪这个问题的相关支持。
在这篇文章中，我们讨论了可实现的解决方法来检测，预防和修复API key是的泄露。我们首先概述了几个检测嵌入在源码中的API keys的方法，
并且用来自于GitHub的样本集评估了它们的有效性。第二，我们枚举了一些机制，它们可以被开发者用于手动地预防或修复源码仓库中API keys的泄露。
最后，我们概述了一个可能的解决方案，结合这些技术来为版本控制系统中的密钥泄露提供工具支持。

0x01 介绍

许多基于web和移动端的应用都是通过Web API来和外部服务相互作用的。而应用与服务之间的认证机制是通过一个API key或者一对API客户端标识符和一个密钥来实现认证。
因为这些服务是打算被服务端应用组件调用的，密钥本身应该理想地待在服务端的程序内存中，所以应用程序的用户是无法访问的。但是，大量应用开发者选择将他们的应用程序源码托管于仓库，比如GitHub和BitBucket。

0x02 密钥泄露检测技术

A. Sample Selection using Keyword Search（用关键字搜索进行样本挑选）

1) 对于窃取泄漏在GitHub上的密钥的攻击者来说，第一个方法，也是最简单的方法就是利用GitHub内置的搜索方法去寻找可能表明密钥存在的关键字或者文件名。

而泄露一些SSH值之后，GitHub在2013年暂停了它的搜索方法。

2) 对于其他类型的密钥，没有必需的文件格式，但是有简单的特征模式。下图是一些服务提供商的API keys。

但是，GitHub不允许对代码进行正则匹配。所以最简单的方式就是对GitHub的每个仓库进行克隆，然后再用正则搜索。但是GitHub上有百万级的仓库，这绝不是一个小事。
但是GHTorrent已经试图提供一个GitHub百万仓库的可查询的镜像，这将使得攻击者更有效地减少可能包含API Keys的样本集。

B. Pattern-based Search（基于规则的搜索）

1) 由于我们的数据集包含可能泄露亚马逊 AWS 证书的项目，我们首先执行一个匹配亚马逊证书规则的全文本搜索。但是一个简单的规则搜索导致了大量的误差。
我们的数据集由利用亚马逊基于Java的Client API组成，我们接下来让我们的搜索仅限于Java源码文件中字符串的字面意思。

2) 我们利用Eclipse JDT的语法分析器从Java源码文件中生成摘要语法树（ASTs,Abstract syntax trees），并且基于字符串字面值节点执行一个规则搜索。这种技术虽然使得误差率有所降低，但仍然有限。

C. Heuristics-driven Filtering（探索式过滤）

1) 有人使用附加的标准去过滤误差项，比如查看在5行内既匹配到Client ID又匹配到Secret Key的实例。可能因为在客户端API声明中或常量文件中它们是硬编码。实验证明，这是一项有效的探索。当这个方法一般精确的时候，它容易错过在证书没有一起闭合时泄露密钥的实例。

2) 另一个方法是去猜测匹配到的API keys是自动生成的还是手写的。我们注意到很多误差项是有手写字符串或者占位符引起的。为了去掉这些想，我们采用一个标准的密码强度估计。
它摒弃那些包含重复字符或字典字词的字符串，接收那些有很高随机性的字符串（熵）。这个技术将检测率提高到91%，余下的误差项是由自动生成的类Key 字符串引起的，它们有不同的用处，比如序列化对象的标识符。

D. Source-based Program Slicing（基于源码的程序切片）

1) 一个重量级的精确查找这些字符串流向客户端API声明的方法是利用程序切片。这里有几个现有技术去精确地跟踪字符串的值，即使它从属于运算，比如级联和字串提取。
然而，这些方法对于我们的目的来说有些过度了，因为我们不希望硬编码密码去经历这些转换。

2) 我们使用了内部开发的一个轻量级的基于源码的流敏感的程序切片算法，这个算法建立了一个系统依赖图，类似于经典的HRB切片。但是它是上下文不敏感的，因为我们不精确匹配调用返回点。

0x03 密钥泄露抑制技术

A. Preventing Leaks Explicitly（明确防止泄露）

1) 不在源码中嵌入密钥，而是放在不被版本控制系统追踪的文件中。

2) 在Git中，使用.gitignore文件来忽略。

3) 包含API keys的文件需要认证共享仓库才能访问。例如工具git-crypt
